# Delivery

## Summary

Wired BLS Quarterly Census of Employment and Wages (QCEW) into the DMV Housing Hub end-to-end: a new ingester pulls per-county federal civilian employment (own_code=1, NAICS 10, agglvl 71) for all 21 DMV counties quarterly back to 2015; the transform step joins it into per-county summaries and emits a suppression-aware DMV-wide aggregate metric file; and the React frontend surfaces the data in three places — a snapshot card and quarterly trend chart on the County page, a "DMV federal jobs" stat on the Home strip, and a real driver card replacing the "Federal job-loss data landing soon" placeholder in the Home page's "What's driving the market" section.

The work was scoped through the CRISPY phases (questions → research → design → outline → plan → implement), executed slice-by-slice as four PR-sized chunks, and shipped as a single PR (#5) since the slices share a tight dependency chain (types → ingester → transform → UI). Out-of-scope cleanup the user flagged mid-implementation — removing the now-stale `FederalExposure` and `ForecastCone` placeholders, fixing source-attribution alignment in `BigChart`, and adding a price-forecast roadmap entry to the README — was rolled into the same PR.

## How to use

### Running the ingester locally
```bash
BLS_API_KEY=... npm run ingest:qcew --workspace=scripts
npm run transform --workspace=scripts
```
The first command writes `scripts/.cache/qcew.json` (~903 observations, ~20 s for the full DMV × quarter sweep). The second joins it into per-county summaries and writes `web/public/data/metrics/federal-employment-dmv.json`.

> Note: `BLS_API_KEY` is referenced by other BLS ingesters but is not strictly required by `data.bls.gov/cew/` (QCEW open data is unauthenticated). The script will run without it.

### Consuming in the frontend
```ts
import { getFederalEmploymentDmv } from './api.js';

const dmv = await getFederalEmploymentDmv();
//   ↳ { total, totalYoY?, asOf, points: [{ date, value }, …], … }

// Per-county already lives on the existing CountySummary:
const county = await getCountySummary('11001');
county.current.federalEmployment;        // latest quarter (number)
county.current.federalEmploymentYoY;     // ratio, e.g. -0.0387
county.current.federalEmploymentAsOf;    // "YYYY-MM-DD"
county.series.federalEmployment;         // MetricPoint[]
```

### CI / production
No manual step. The monthly cron in `.github/workflows/ingest.yml` runs `--all`, which now picks up `qcew` via the registry. Cloudflare Pages redeploys on each commit to `main`.
